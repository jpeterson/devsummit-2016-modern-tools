Typescript is a superset of JavaScript, providing features on top of JavaScript that can help you to code more efficiently. Typescript has been around for a few years, and in the early days was a competitor to Google's Dart. Google's Dart was going to be a NEW web programming language though, where Typescript took a less radical approach than dart - again - where it's totally compatible with plain JavaScript out-of-the-box.

Many projects and companies are using Typescript. Sitepen is using Typescript to develop Dojo 2, the new version of Dojo. This doesn't mean you must develop your applications in Typescript -- simply that they are writing the underlying toolkit code in Typescript. AngularJS, a popular JavaScript Framework, has chosen to focus on the Typescript user for Angular version 2, which is currently in beta and should be out soon - so unlike Dojo, if you're going to use Angualar2, you pretty much need to be using Typescript.

TypeScript Features

I'll now go through a few of the features of TypeScript, so you can get a sense of what benefits it'll bring if you use it on one of your projects.

Type Annotations


function buildName(firstName, lastName) {
    return firstName + " " + lastName;
}

Becomes:


function buildName(firstName: string, lastName: string) {
    return firstName + " " + lastName;
}
All of the ES2015 codes styles we've been showing so far still apply in the TypeScript world. But you can add things to the code that help you. One example is type annotations on functions. This is actually where TypeScript gets it's name - we're taking JavaScript, a language where variable and function return types are NOT declared, and we're using types. The compiler enforces the types that you set. The syntax is quite simple - you can see in this example, you just put a colon after the function argument name, and put what type it is. The types that TypeScript supports include boolean, number, string, arram, enum, "any", and void. Let's see these types in action [DEMO].

Interfaces

DUCK IMAGE

One of TypeScript's core principles is that type-checking focuses on the 'shape' that values have. This is called "duck typing", named after the classic expression "If it looks like a duck, swims like a duck, and quacks like a duck, then it probably is a duck." So if Typscript is just looking at the "shape" of values when determining type, naming them would make things easier. That's what interfaces are.

Interfaces


function greeter(person : {firstname: string, lastname: string}) {
    return "Hello, " + person.firstname + " " + person.lastname;
}

var user = {firstname: "Jack", lastname: "Dangermond"};

document.body.innerHTML = greeter(user);

Becomes:


interface Person {
    firstname: string;
    lastname: string;
}

function greeter(person : Person) {
    return "Hello, " + person.firstname + " " + person.lastname;
}

var user = {firstname: "Jack", lastname: "Dangermond"};

document.body.innerHTML = greeter(user);

Classes


class President {
    fullname : string;
    constructor(public firstname, public middleinitial, public lastname) {
        this.fullname = firstname + " " + middleinitial + " " + lastname;
    }
}

interface Person {
    firstname: string;
    lastname: string;
}

function greeter(person : Person) {
    return "Hello, " + person.firstname + " " + person.lastname;
}

var user = new President("Jack", "A.", "Dangermond");

document.body.innerHTML = greeter(user);
Classes work much the same as ES2015 classes, but TypeScript allowed their usage much earlier than transpilers caught up to the game. In our class, President, we have a member property, 'fullname', and a constructor. We're referring to the member property using the 'this' keyword inside the constructor. All typical Object Oriented patterns.

Inheritance


class President {
    fullname: string;
    constructor(public name) {
        this.fullname = name;
    }
}

class EsriPresident extends President {
    constructor(public name, public favoriteMapProjection) {
        super(name);
    }
}


function greeter(person: EsriPresident) {
    return "Hello, " + person.fullname + ". " + person.favoriteMapProjection + " - nice projection!";
}

var user = new EsriPresident("Jack", "WebMercator");

document.body.innerHTML = greeter(user);

Global State
